GET V/S POST HTTP Request Methods

The HTTP (Hyper Text Transfer Protocol) is designed to communicate between client and servers. HTTP works as request-response protocol between client and server.

HTTP defines a set of request methods which tell about desired action to be performed for each request. The request methods are sometimes also referred as HTTP verbs.

GET METHOD: GET is used to request data from specified resource. It is one of the most common HTTP methods. A query string having name-value pairs is sent in URL of GET request.

GET request can be cached. They remain in browser history and can also be bookmarked. GET request should never be used when dealing with sensitive data. They have length restrictions and can only be used to request data, not to modify it.

POST METHOD: POST method is used to send data to a server to create/ update a resource. The data which is sent to the server with POST request, is stored in the request body.
